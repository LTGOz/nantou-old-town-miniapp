# 南头古城 · 青年兴趣社交小程序

## 项目概述

以南头古城青年租户为核心服务对象，打造轻量化微信小程序，核心功能包括活动聚合展示、青年兴趣社交圈子，以及创新的**场所入座率实时查询**模块。帮助古城青年租户解决信息碎片化、社交闭塞、场所拥挤度未知等核心痛点。

Slogan：**兴趣相聚，轻社交，遇古城同频青年**

## 功能模块

### 1. 首页
- 古城沉浸式 Header（渐变背景 + 日期/Slogan）
- Banner 轮播（社区资讯热门推荐）
- 入座率快捷入口（咖啡馆 / 餐厅 / 图书馆三分类）
- 活动分类导航（社区 / 商户 / 个人发起）
- 热门活动卡片推荐
- 热门商户网格 + 入座率预览

### 2. 场所入座率查询 ★ 核心模块
- **实时状态概览**：顶部统计栏（总商户数 / 空闲 / 适中 / 较满数量）
- **分类筛选**：按商户类型（全部 / 咖啡馆 / 餐厅 / 图书馆）
- **状态筛选**：按入座率水平（全部 / 空闲<40% / 适中40-70% / 较满>70%）
- **可视化展示**：
  - 百分比进度条 + 红/橙/绿三色热力标识
  - 空余座位数实时显示
- **时段趋势图**：柱状图展示当日各时段入座率变化
- **底部面板**：选中商户后展示实时入座率详情 + 趋势图
- **一键导航**：调用微信地图打开商户位置

### 3. 活动中心
- 三类活动分区（社区活动 / 商户活动 / 个人自发）
- 费用标签、报名进度条
- 活动详情页（资料、进度、报名入口）

### 4. 古城圈子
- 热门兴趣社群（摄影 / 桌游 / 读书 / 咖啡 / 徒步 / 手作）
- 邻里互助广场（转租 / 交换 / 互助）
- 社群加入与互动入口

### 5. 我的
- 个人中心（头像 / 昵称 / 租户认证标识）
- 我的报名 / 收藏 / 发布 / 积分
- 消息中心 / 设置 / 帮助入口

## 本地开发

### 环境要求

- npm 或 yarn

```bash

# 构建产物位于 dist/ 目录
```

## 部署发布流程

### 微信公众平台提审步骤

1. **注册小程序账号**
   - 前往 [微信公众平台](https://mp.weixin.qq.com/) 注册小程序账号
   - 完成主体认证（企业/个人），获取 AppID

2. **配置 project.config.json**
   ```json
   {
     "appid": "你的小程序AppID",
     "projectname": "南头古城青年社交",
     "setting": { "es6": true, "minified": true }
   }
   ```

3. **代码检查与优化**
   - 确保所有 mock 数据替换为真实 API 接口
   - 移除 console.log 调试日志
   - 启用代码压缩（生产构建自动处理）

4. **构建生产包**
   ```bash
   npm run build:weapp
   ```
   构建产物自动输出到 `dist/` 目录

5. **上传代码**
   - 打开微信开发者工具
   - 导入项目，选择 `dist/` 目录
   - 点击「上传」，填写版本号与变更说明

6. **提交审核**
   - 登录微信公众平台后台
   - 进入「版本管理」→ 选择已上传版本 →「提交审核」
   - 填写功能演示说明（重点描述入座率查询等核心功能）
   - 提交审核类目建议：**社交 > 社区/论坛** 或 **工具 > 信息查询**

7. **审核通过后发布**
   - 审核通过后手动点击「发布」上线

## Mock 数据说明

开发阶段使用本地 Mock 数据，位于 `src/data/` 目录：

| 文件 | 数据内容 | 条数 |
|------|---------|------|
| merchants.ts | 古城商户（含坐标、评分、坐席数） | 12 条 |
| activities.ts | 三类活动（社区/商户/个人） | 10 条 |
| occupancy.ts | 入座率数据（实时 + 分时段趋势） | 12 条 |

**对接真实 API 后**，替换 `src/services/` 下的服务函数即可，前端页面代码无需修改。
